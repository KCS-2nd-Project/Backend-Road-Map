Elastic search

검색 및 분석 엔진으로, 대용량 데이터의 실시간 검색 및 분석을 위해 설계된 오픈 소스 분산 검색 엔진이다. Elasticsearch는 Apache Lucene을 기반으로 하며, RESTful API를 사용하여 데이터를 저장, 검색, 분석할 수 있습니다. 주로 로그 및 이벤트 데이터, 웹 애플리케이션 검색, 비즈니스 인텔리전스 등 다양한 용도로 사용된다.

Elasticsearch는 검색을 위해 단독으로 사용되기도 하며, ELK(Elasticsearch / Logstatsh / Kibana) 스택으로 사용되기도 한다.

ELK(Elasticsearch / Logstatsh / Kibana) 스택

ELK는 접근성과 용이성이 좋아 최근 가장 핫한 Log 및 데이터 분석 도구이다.

  • Lojstash : 다양한 소스(DB, CSV 등)의 로그 또는 트랜잭션 데이터를 수집, 집계, 파싱하여 Elasticsearch 로 전달한다.
  • Elasticsearch : Logstatsh로 부터 받은 데이터를 검색 및 집계를 하여 필요한 관심 있는 정보를 획득한다.
  • Kibana : Elasticsearch의 빠른 검색을 통해 데이터를 시각화 및 모니터링 한다.

특징 및 장점

  • 오픈소스 검색엔진이다.
  • Lucene 을 기반으로 구축되어 Full-text 검색에 뛰어나다.
  • 역색인 구조로 특정 문서의 위치 찾는데 빠르다.
  • 데이터 CRUD 작업을 HTTP Restful API를 통해 수행할 수 있다.
  • 속도, 확장성, 복원력 뿐 아니라, 효율적으로 저장하고 검색할 수 있게 해준다.
  • Kibana와 연동하여 실시간 시각화를 제공해준다.
  • 죽은 노드를 감지하고 삭제하여 일부 노드에 문제 발생하여도 문제 없이 서비스를 제공해준다

단점

  • 빠르지만 완전한 실시간이 아니다.
  • 트랜잭션과 롤백 기능이 없다
  • 데이터 UPDATE를 제공하지 않는다.
