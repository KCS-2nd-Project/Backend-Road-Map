GitHub & GitLab

  • GitHub : https://github.com/
  • GitLab : https://about.gitlab.com/

  • GitHub의 사용자 수는 약 5,600만 명인 반면 GitLab의 사용자 수는 3,100만 명 이상
  • Java 개발자의 35% 이상이 다른 코드 저장소 대신 GitLab을 선택

GIT

  • 코드에 사소한 변경이 있는 경우에도 전체 코드베이스의 스냅샷이 생성한다.
  • 개발자는 이제 필요한 경우 쉽게 이전 코드 버전을 검토하거나 돌아갈 수 있다. 
  • 이는 코드 품질과 코딩 표준을 향상시키기 위한 필수 요소

Git을 사용하면 VCS 서비스에 가입할 필요 없이 자체 VCS 서버를 만들 수 있다.

Git을 사용하면 전 세계에 분산된 파트너 또는 팀과 함께 IT 프로젝트를 원활하게 실행할 수 있다.

GitLab

→ 개발자가 코드를 원활하게 테스트, 모니터링 및 배포할 수 있도록 해주는 클라우드 기반 Git 리포지토리이자 DevOps 플랫폼

GitLab의 특징

  • Bitbucket 또는 Google Code에서 가져온 코드 저장소
  • 커뮤니티 에디션 오픈 소스 저장소 플랫폼
  • 서버 저장소의 원활한 유지 관리
  • 강력한 UI 및 인증 기능

GitLab의 장점

  • 오픈소스 라이센스
  • 모든 계획에 셀프 호스팅 기능 제공
  • 사용자 친화적인 패키지 배포 서비스
  • 지원되는 CI/CD 수명주기

GitLab의 단점

  • 소프트웨어 수명주기 전반에 걸쳐 버그 관리
  • 일부 엔터프라이즈 수준 소프트웨어 개발 기능이 없음

GitHub

GitHub의 특징

  • 지점을 비교하고 완전한 비교 보기 가능
  • GitHub는 웹사이트 게시 또는 호스팅 가능
  • 클라우드 호스팅 또는 버그 추적을 위한 타사 API 통합

GitHub의 장점

  • 쉬운 문서 공유
  • 강력한 오픈 소스 코드베이스 구조
  • 타사 제공업체와 빠른 통합
  • SVN, HG 및 TFS에 대한 광범위한 지원

GitHub의 단점

  • 효율적인 REST 및 RESTful API 개발 부족
  • 다양한 개발 기능 부족
  • 유료 버전 존재

GitLab & GitHub 유사점

타사 통합 서비스둘 다 많은 타사 통합과 함께 제공
명확한 라벨링 방식둘 다 간소화된 소프트웨어 개발 모니터링 관행을 가능하게 하기 위해 깔끔한 라벨링 체계를 사용
CI / CD 서비스CI/CD 서비스는 GitHub에서 타사 앱과 함께 사용 가능. GitLab은 사전 구성
코드 변경 사항 미리보기두 플랫폼 모두에서 코드 변경 사항 미리보기가 가능
Wiki 기반 문서GitHub와 GitLab은 모두 Wiki 기반의 광범위한 문서를 제공
팀 토론둘 다 팀 토론이 가능
프로젝트 관리 대시보드둘 다 프로젝트 관리 대시보드를 지원
부하 성능 테스트GitHub는 앱에서 로드 성능 테스트 기능을 제공
GitLab은 유료 요금제를 제공

GitLab & GitHub 차이점

  • 두 개의 온라인 코드 저장소 서비스는 서로 다른 두 회사에서 제공

GitLabGitHub
인증 수준다양한 역할과 책임에 따라 액세스 및 권한을 쉽게 설정 & 수정 가능팀 구성원에게 읽기 또는 쓰기 액세스 권한을 부여할지 여부를 결정
CI/CD 서비스내장 CI를 무료로 제공 & DevOps 자동화 솔루션을 제공CI/CD 서비스를 활용하려면 Heroku와 같은 추가 앱이 필요
가져오기 및 내보내기다른 공급업체에서 데이터를 가져오거나 내보내는 방법에 대한 기본 문서를 제공인기 있는 Git 리포지토리에 대한 자세한 문서를 제공하지 않음
작업 흐름작업 흐름 안정성을 강조작업 흐름 속도를 강조
오픈 소스커뮤니티 에디션용 오픈 소스오픈 소스가 아닌 클로즈 코어
프로젝트 분석사용자는 프로젝트 개발 차트를 검토 가능사용자는 프로젝트 분석을 얻을 수 없지만 커밋 기록을 검토 가능
