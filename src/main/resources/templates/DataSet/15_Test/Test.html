Test

Test Code 란?

  • Test Code를 통해 software의 기능과 동작의 결함을 사전에 발견하고 수정함으로써, 실제 사용자가 소프트웨어를 사용하는 중에 문제가 발생하는 것을 방지할 수 있다.
  • 클래스간의 결합도와 의존성을 테스트 할 수 있다.
  • 개발자간 협업을 원활하게 하고 유지 보수에 필요한 문서화 작업을 줄일 수 있다.

<출처 : https://www.geeksforgeeks.org/software-engineering-sdlc-v-model/>

Unit Test

  • 개별적인 코드 단위(메서드)가 의도한 대로 동작하는지 확인하는 과정
  • 각각의 코드 단위가 정확한 입력값과 출력값을 반환하는지 확인

Integration Test

  • 서로 다른 모듈 간의 상호작용을 테스트하는 과정
  • 모듈 간 인터페이스 테스트, 시스템 레벨 테스트 등의 방법으로 수행

Functional Test

  • 어떤 어플리케이션이 제대로 동작하는지 완전한 기능을 테스트하는 것

테스트 코드 작성의 이점

  1. 어떤 애플리케이션을 만드는지 정확한 인지 : 테스트 코드 작성을 통해 요구사항 기능을 정리하고 코너 케이스 (Corner case)를 찾고 문서화 가능
  1. 리팩토링 진행시 부담 덜기 : 코드 수정 후에도 기능이 정상적으로 작동하는지 검증할 수 있으므로 개발자의 불안감 감소
  1. 결합도와 의존성이 낮은 코드 지향 : 의존성이 높은 부분을 찾고 개선함으로써 프로젝트 코드 품질 향상

Test 작성에 자주 사용하는 라이브러리

  1. AssertJ : 자바 테스트를 돕기 위해 메서드 체이닝을 통해 읽기 쉬운 테스트 코드 작성 가능
    • 참고 : https://assertj.github.io/doc/

  1. Junit : 자바 프로젝트를 위한 단위 테스트 프레임워크
    • 참고 : https://junit.org/junit5/

  1. Mokito : 자바 오픈소스 테스트 프레임워크, 실제 객체를 모방한 가짜 객체 (Mock 객체)를 생성하여 편리한 테스트가 가능
    • 참고 : https://site.mockito.org/
    • 참고 : https://www.nextree.io/mockito/
