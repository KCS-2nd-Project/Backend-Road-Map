RDB

PostgreSQL

특징

  • 라이선스에 대한 비용 문제가 없음 : BSD 라이선스이며, 라이선스의 가장 큰 특징은 소스를 변경하고 그 소스를 숨긴 채 재배포 해도 법적으로 문제가 없다는 점
  • 오래된 오픈소스의 안정성 : 매우 가볍게 돌아가는 데이터베이스이며 대용량 데이터의 처리에도 큰 문제점이 발견되지 않았고, 표준SQL 잘 따름
  • 여전히 발전중인 데이터베이스 : 빠른 속도로 업데이트가 진행됨
  • 장점이자 단점인 독창적인 자료형 및 문법을 사용하여 국내 레퍼런스가 적음 : postgreSQL의 최근 버젼들이 오류메시지를 한글화해서 보여주면서, 한글 오류를 해외 레퍼런스에서 검색하는데 있어 많은 어려움을 겪기도 했다.

MySQL

특징

  • MySQL 은 현재 2위로 많이 사용하는 데이터베이스 관리 시스템이다.
    • 참고) - DB-Engines Ranking
  • C, C++로 만들어졌으며 MyISAM 인덱스 압축 기술, B-tree 기반의 인덱스, Thread 기반의 메모리 할당 시스템, 매우 빠른 조인, 최대 64개의 인덱스를 제공
  • 대용량 데이터베이스를 위해 설계되어 있고 롤백, 커밋, 이중 암호 지원 보안등의 기능을 제공

Oracle

특징

  • 미국의 오라클 회사에서 제작한 세계 점유율 1위 데이터베이스 관리 시스템
    • 참고 : https://db-engines.com/en/ranking

오라클 장점

  • 관리 시스템 : 멀티플 데이터베이스 튜닝이 가능하고 다수의 사용자가 동시에 접근이 가능
  • 변화 관리 : 변경 plan을 작성하고 실제 구현하기 전에 변경 사항의 효과 볼 수 있음.
  • 경고 : 오류가 발생하면 설정되어 있는 계정 및 이메일로 연락이 온다. 경고는 예정된 가종 정지 시간 동안 차단
  • 분산 처리: DBMS 실행 컴퓨터, 서버 역할 컴퓨터, DB 응용 프로그램 실행 컴퓨터 각각 다르게 분산처리가 가능
  • 용량과 처리 : 다른 데이터베이스보다 고성능의 트랜잭션을 처리가능

오라클의 단점

  • 비용적인 부담이 있음
  • 기능이 많기 때문에 초보자가 사용하기 어려움이 있음
  • 높은 지원의 하드웨어 사양이 필요
