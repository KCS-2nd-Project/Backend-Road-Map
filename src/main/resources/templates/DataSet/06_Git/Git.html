Git

Git이란?

Git은 여러 명의 사용자들간 파일 관리에 관한 작업을 조율하기 위한 스냅샷 스트림 기반의 분산 버전 관리 시스템이다.

팀 프로젝트에서 여러 사람이 작업을 진행할 때, 소스 코드를 효과적으로 관리하기 위한 도구 파일이 생성, 수정, 삭제 될 때 Git은 해당 파일을 관리 대상으로 인식한다. 파일의 변경사항을 추적하여 여러 사용자들이 파일에 관한 작업을 조율하는 것을 돕는다.

repository(저장소)

프로젝트의 전체 기록이 저장되는 저장 공간이다. 로컬 저장소와 원격 저장소로 나눌 수 있다.



branch

  • 코드를 독립적으로 개발하기 위한 분기점이다.
  • 기능을 추가하거나, 버그를 수정할 때마다 새로운 브랜치를 만들어 작업한다. 브랜치들은 나중에 합칠 수 있다.

add, commit, push

로컬 저장소의 변경점을 원격 저장소에 추가하기 위해서는 다음의 세 가지 작업을 거쳐야한다.

git add .
git commit
git push

1. add

  • Staging Area에 변경 내용을 추가한다.
  • 각 파일이나 디렉토리를 지정하여 하나씩 추가하거나, git add . 명령어를 통해 한 번에 추가할 수 있다.

2. commit

  • Staging Area의 변경 이력을 파일에 기록
  • 각각의 커밋은 해시 값으로 식별되며, 하나의 커밋이 한 개의 변경 이력으로 작용한다.
  • 커밋과 함께 적절한 커밋 메세지를 작성하여 무엇에 관한 변경점인지 알기 쉽도록 할 수 있다.

3. push

  • 로컬 저장소에 커밋된 변경점들을 원격 저장소로 업로드

pull

git pull
  • 원격 저장소의 변경사항을 로컬로 복사한다.
  • 만일 원격 저장소에 변경 사항이 존재하는 상황에서 pull 명령을 실행할 경우, 현재 브랜치에서 새로 작업한 내용이 손실되는 일이 생길 수 있으니 주의해야한다.
  • pull 명령을 사용하기 전 featch 명령을 통해 원격 저장소의 변경 사항을 먼저 확인하는 것을 권장한다.

log

git log
  • 로컬 저장소의 커밋 내역을 확인할 수 있다.

merge, rebase

merge와 rebase를 통해 브랜치를 하나로 합칠 수 있다.

아래 그림과 같은 두 브랜치가 합쳐지는 과정을 살펴보자.

1. merge

git checkout main
git merge feature
  • 개별 브랜치의 변경 사항을 하나로 병합한다.
  • 이전 브랜치에 존재하던 변경 사항은 사라진다.

2. rebase

git checkput feature
git rebase main

git checkout main
git merge feature
  • 대상 브랜치를 main 브랜치를 대상으로 재정렬한다.
  • 이후 merge 명령어를 사용한다면 이전 main 브랜치에 존재하던 변경점을 유지한 채로 브랜치를 합칠 수 있다.
