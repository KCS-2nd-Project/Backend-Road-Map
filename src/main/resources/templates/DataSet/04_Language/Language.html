JAVA

https://dev.java/

Java란?

C언어에 객체 지향적 기능을 추가하여 만든 객체지향 언어로 개발된 프로그래밍 언어이다.

Java의 특징

  • Java 컴파일러는 바이트코드인 클래스파일(.class)을 생성하고, JVM은 바이트코드를 기계어로 바꾸어 실행한다.
  • 웹 및 모바일 앱 개발에 주로 사용된다.

Java의 장점

  • Java 가상 머신(JVM, Java Virtual Machine)을 사용해, 운영체제와는 독립적으로 동작할 수 있다.
  • 불필요한 기능을 제거해 다른 언어에 비해 배우기가 쉽다.
  • 자동 메모리 관리 등을 지원해 다른 언어에 비해 안전성이 높다.
  • 연산자 오버로딩을 금지하고 제네릭을 도입함으로써 코드의 가독성이 높다.

Java의 단점

  • Java는 실행을 위해 자바 가상 머신을 거쳐야 하므로, 다른 언어에 비해 실행 속도가 느리다.
  • 자바는 예외 처리가 잘 되어 있지만, 개발자가 일일이 처리를 지정해 줘야 한다는 불편함이 있다.
  • 자바는 다른 언어에 비해 작성해야 하는 코드의 길이가 긴 편이다.

C#

https://learn.microsoft.com/ko-kr/dotnet/csharp/

C#이란?

Microsoft에서 개발한 객체지향 프로그래밍 언어로, 주로 Windows 플랫폼에서 사용되며, 최근에는 크로스 플랫폼 지원이 강화되고 있다.

C#의 특징

  • .NET 프레임워크를 기반으로 하여 견고하고 보안성이 높은 프로그램 제작이 가능하다.
  • Window 프로그램은 물론 모바일, 프로그램, 클라이언트-서버 프로그램, 데이터베이스 프로그램, 웹 기반 프로그램, 사물인터넷(IOT) 프로그램 등 다양한 종류의 어플리케이션 개발에 사용 가능하다.

C#의 장점

  • 객체 지향 프로그래밍을 지원해 코드의 재사용성이 높고 유지보수가 용이하다.
  • 자동 메모리 관리 기능으로 메모리 누수를 예방할 수 있어 메모리 관리 및 성능 향상에 유용하다.
  • .NET 프레임워크 기반으로 다양한 실행될 수 있다.
  • .NET Core를 이용하여 리눅스, 맥 OS 등 다양한 운영체제에서도 실행 가능하다.

C#의 단점

  • 실행을 위해 자바 가상 머신과 유사한 .NET 런타임이 필요하므로 시작 속도가 느릴 수 있다.
  • 복잡한 메모리 관리를 요구한다.

Rider, VS Code, VS Studio, Mono Develop

Python

https://www.python.org/

Python이란?

Python은 간결하고 읽기 쉬운 문법을 가진 고수준, 인터프리터 언어로, 다양한 프로그래밍 스타일을 지원한다.

Python의 특징

  • 컴파일 과정 없이 인터프리터가 소스 코드를 한 줄씩 읽어 들여 곧바로 실행하는 스크립트 언어이다.
  • 동적 타입 언어로 변수의 자료형을 지정하지 않고 단순히 선언하는 것만으로 값을 지정할 수 있다.
  • 리눅스(Linux), 유닉스(Unix), 윈도우즈(windows), 맥(Mac) 등 대부분의 운영체제에서 모두 동작한다.
  • 데이터 사이언스 및 머신러닝 등의 분야에서 사용된다.

Python의 장점

  • 쉬운 문법으로 다양한 도메인에서 사용 가능하며, 강력한 커뮤니티와 생태계를 갖추고 있다.
  • 쉽고 간결한 문법으로 높은 생산성을 가지며, 복잡한 구문으로 인한 오류 발생을 줄이고 빠른 개발이 가능하다.
  • 대표적인 글루(Glue) 언어에 해당하며 다른 언어나 라이브러리에 쉽게 접근해 연동 가능하다.

Python의 단점

  • 실행 속도가 컴파일 언어에 비해 상대적으로 느리다.
  • 빠른 속도를 요구하거나 하드웨어를 직접 컨트롤 하는 프로그램에는 적합하지 않다.

IDE & Text Editer

IDLE, PyCharm, VS Code, jupyter

PHP

https://www.php.net/

PHP란?

PHP는 ‘Personal Hypertext Tools’의 약자로 웹사이트 제작에 특화된 벡엔드(서버) 언어로 동적 웹 페이지 생성 및 데이터베이스 상호작용을 위한 내장 지원을 제공한다.

PHP의 특징

  • 주로 웹 개발에 특화되어 있으며, 빠른 개발이 가능하다.
  • 서버 측에서 실행되므로 클라이언트에게 코드가 노출되지 않는다.

PHP의 장점

  • 다른 언어에 비해 직관적이다.
  • 결과값을 바로 받아올 수 있다.
  • 대부분의 운영체제에서 구현 가능하며 다양한 호스팅 환경에서 지원된다.
  • 비용이 발생하지 않아 쉽게 접근 가능하다.

PHP의 단점

  • 내장 함수나 인자 이름 규칙의 일관성이 없다.
  • 언어의 발전이 느리고, 모듈화 및 테스트가 어렵다.

IDE & Text Editer

PhpStorm, eclipse, VS Code

Ruby

https://www.ruby-lang.org/en/

Ruby란?

Ruby는 객체지향 및 동적 타이핑 언어로, 간결하면서도 효율적인 문법을 가진 언어이다.

Ruby의 특징

  • 생산성을 높이기 위한 문법과 메타프로그래밍 기능이 있다.
  • 완전한 객체 지향 프로그래밍 언어이다.
  • Rails 프레임워크를 통해 웹 개발이 지원된다.
  • 웹 앱 개발부터 데이터 분석에 사용된다.

Ruby의 장점

  • 간결하고 읽기 쉬운 문법을 가지고 있어 생산성을 높여준다.
  • 동적 타입언어로서 변수의 타입을 미리 선언할 필요가 없어 코드를 더 유연하게 작성할 수 있게 한다.
  • 메타프로그래밍 기능을 제공하여 개발자는 실행 중에 프로그램의 동작을 변경하거나 확장 가능하다.

Ruby의 단점

  • 실행시간에 타입 체크가 이루어지기 때문에 일부 언어보다 성능 면에서 느릴 수 있다.
  • 동적 타입과 객체 지향적 특성으로 인해 메모리 사용량이 크다.
  • 다중 코어 시스템에서 병렬 처리를 제대로 활용하기 어렵다. 따라서 CPU 집약적인 작업에는 적합하지 않을 수 있다.
  • 문법과 관계에 익숙해지는데 어렵다.

IDE & Text Editer

RubyMine, VS Code, Sublime Text ,Atom

Go

https://go.dev/

Go란?

Go는 Google에서 개발한 컴파일 언어로, 간결한 문법과 빠른 컴파일 속도, 병렬 프로그래밍을 위한 지원 등을 특징으로 한다.

Go의 특징

  • 정적 타이핑, 간결한 문법, 병렬 프로그래밍을 위한 강력한 지원이 있습니다.
  • 빠른 컴파일 속도와 효율적인 메모리 관리를 제공합니다.
  • 자체적인 스케줄러에 의해 관리되는 경량 쓰레드이며, OS에서 관리하는 경량 쓰레드 보다 가볍다.

Go의 장점

  • 실행 속도가 매우 빨라 대규모 시스템에서 높은 성능을 제공한다.
  • 문법이 간결하고 사용하기 쉬워 코드 작성이 편리해지고 가독성이 높다.
  • 가벼운 쓰레드 모델을 제공하여 동시성 처리가 용이하다.
  • 표준 라이브러리가 다양하고 풍부하여 다양한 작업에 필요한 기능을 제공한다.

Go의 단점

  • 상대적으로 새로운 언어이기 때문에 다른 언어들에 비해 생태계가 부족한 편이다. 따라서 특정한 라이브러리나 도구를 찾기 어렵다.
  • 몇 가지 제어구조를 간소화하고 제한적으로 만들어 코드의 유연성을 제한할 수 있다.

IDE & Text Editer

GoLand, VS Code

Rust

https://www.rust-lang.org/

Rust란?

Rust는 Mozilla에서 개발한 시스템 프로그래밍 언어로, 메모리 안정성과 성능에 중점을 둔 언어이다.

Rust의 특징

  • 메모리 안정성을 보장하면서 C와 유사한 성능을 제공한다.
  • 많은 공개 프로젝트에서 사용되고 있으며, 시스템 프로그래밍, 웹 서버 및 브라우저, 게임 개발, 블록체인, IoT 등 다양한 분야에서 사용된다.

Rust의 장점

  • 안전한 메모리 관릴르 보장한다.
  • C와 C++와 같은 저수준 언어와 유사한 고성능을 보인다. 또한, 속도는 뛰어나며, 메모리 사용량이 적다.
  • Rust의 컴파일러는 빠르게 동작하며, 다른 언어에 비해 빠른 빌드 속도를 가지고 있다.
  • 패턴 매칭, 제네릭, 매크로, 클로저 등 다양한 기능을 활용해 높은 생산성을 제공한다.

Rust의 단점

  • C와 C++와 같은 저수준 언어와 유사하지만, 새로운 개념과 구문이 많아 학습 곡선이 높다.
  • 개발자들이 참고할 수 있는 온라인 문서가 부족하다.

JavaScript

JavaScript란?

JavaScript는 웹 브라우저에서 동작하는 스크립트 언어로, 주로 웹 개발에 사용된다.

JavaScript의 특징

  • 동적 타이핑, 이벤트 기반 프로그래밍을 지원하며, 주로 클라이언트 측 웹 개발에 사용된다.
  • Node.js를 통해 서버 사이드에서도 실행이 가능하다.
  • 객체 기반의 언어이다.

JavaScript의 장점

  • 컴파일 과정이 필요 없으므로, 빠른 시간 안에 스크립트 코드를 작성할 수 있다.
  • 웹 브라우저에서 기본적으로 실행되며, 다양한 라이브러리와 프레임워크를 제공한다.
  • 다양한 플랫폼에서 실행 가능하며, 비동기 프로그래밍을 지원한다.

JavaScript의 단점

  • 동적 타이핑으로 인한 버그 발생 가능성이 있다.
  • 다양한 브라우저 간의 호환성 문제가 발생할 수 있다.
  • 한정된 객체와 객체 함수를 제공한다.

IDE & Text Editer

VS Code, WebStorm, Brackets
