OS

OS work

OS(Operating System)란?

운영 체제는 사용자가 컴퓨터 하드웨어간의 인터페이스를 제공해 주는 프로그램이다. 하드웨어와 소프트웨어의 리소스를 관리하고 컴퓨터 프로그램을 위한 공통 서비스를 게공하여 소프트웨어가 원활하게 동작할 수 있도록 한다.

운영체제의 기능

  • 컴퓨터 자원 관리
    • 프로세스 관리: 프로세스의 생성, 제거, 재실행, 프로세스와 스레드의 스케줄링 등
    • 메모리 관리: 메모리 공간 할당 및 회수, 메모리 추적 등
    • 저장장치 관리: 파일 및 디렉토리의 생성 및 제거, 디스크 관리, 캐시 관리 등
  • 하드웨어의 고장 탐색 및 오류 처리
  • 컴퓨터 자원 보호 및 보안 관리
  • 사용자에게 편리한 인터페이스 제공

운영체제의 종류

운영체제는 다양한 종류가 있으며 그중 대표적인 운영체제들은 다음과 같다.

  • 윈도우(Windows)
  • 맥 OS(Mac OS)
  • 리눅스(Linux)
  • 유닉스(Unix)
  • 우분투(Ubuntu)

Process 관리

Process란?

디스크에 저장되어 있던 실행 가능한 프로그램 파일이 주기억 장치에 적재되어 운영체제의 제어를 받고 있을 때 해당 프로그램을 프로세스 라고 한다. 다시 말해, 프로세스는 CPU에 의해서 현재 실행되고 있는 프로그램을 의미한다.

프로세스 스케줄링이란?

프로세스 스케줄링은 프로세스들 사이의 우선순위를 관리하고 시스템의 자원을 해당 프로세스에게 할당하는 작업을 의미한다. 이는 CPU 사용 시간을 프로세스간에 효율적으로 할당하여 CPU 이용률을 증가시키고 응답시간, 반환시간, 대기시간, 오버헤드를 최소화시키는 것을 목적으로 한다.

선입선처리(FCFS, First Come First Served) 스케줄링

선입선처리 스케줄링은 CPU에 먼저 도착하는 순서대로 프로세스를 할당해 실행하는 방식이다.

알고리즘이 간단하고 구현하기 쉽지만, 하나의 긴 프로세스가 있다면 해당 프로세스로 인해 나머지 프로세스가 긴 대기시간을 가져 CPU 효율성이 낮아진다. 또한 실행 시간이 짧은 작업이나 중요한 작업을 오랫동안 기다리게 할 수 있다.

최단 작업 우선(SJF, Short Job First) 스케줄링

최단 작업 우선 스케줄링은 작업이 끝나기까지의 실행 시간 추정치가 가장 작은 작업을 먼저 실행시키는 방식이다.

선입선처리 스케줄링 방식보다 평균 대기시간이 작다. 하지만 실행 시간이 긴 프로세스의 경우 우선 순위가 계속해서 뒤로 밀려나는 기아 현상에 빠져 무한 대기 상태가 발생할 수 있다.

라운드 로빈(Round Robin) 스케줄링

라운드 로빈 스케줄링은 동일한 시간 할당량을 사용하는 시분할 처리 방식이다. 시간 할당량 안에 작업을 마치지 않으면 준비 대기 리스트의 가장 뒤로 배치되어 다시 차례를 기다린다.

시간 할당량이 클 수록 선입선처리 방식과 유사해지고, 시간 할당량이 적을수록 그만큼 프로세스간의 교체 수와 시간이 늘어나 오버헤드가 증가한다.

Threads

Threads란?

스레드는 CPU이용의 기본 단위로, 프로세스 내에서 실행되는 독립적인 작은 실행 단위를 뜻한다.

일반적으로 한 프로그램은 하나의 스레드를 가지고 있지만, 프로그램 환경에 따라 다수의 스레드가 하나의 프로세스 내에서 동시에 실행될 수도 있다.

Multithreading(멀티스레딩)

각 스레드는 프로세스 내에서 실행 경로를 나타내며, 자체 경로를 가지지만, 다른 스레드와 메모리 공간을 공유한다. 즉, 공유 메모리나 자원을 활용할 수 있다. 스레드를 병렬 처리하면 여러 스레드가 동시에 실행되어 여러 작업을 동시에 처리할 수 있는데 이를 멀티스레딩(Multithreading)이라고 한다

멀티스레딩은 동시성이나 병렬성 방식으로 수행된다.

  1. Concurrnrtcy(동시성):
    • 여러 개의 스레드가 하나의 코어에서 번갈아가면서 실행되는 성질을 말한다.
    • 여러개의 스레드를 모두 동일한 정도로 실행시킨다.
    • 멀티 코어에서 멀티 스레드를 사용하여 동시성을 만족할 경우, 실제 물리적 시간으로도 동시에 실행된다.

  1. Parallenlism(병렬성):
    • 한 개 이상의 스레드를 포함하는 각 코어들이 동시에 실행되는 성질을 말한다.
    • 스레드들이 실제로 동시에 수행되는 것을 의미한다.
    • 동시성을 만족하여도 병령성을 만족하지 않을 수 있다.

Basic Termainl Commands

1. 파일 시스템 탐색 명령어

1-1. pwd: 현재 작업중인 디렉토리의 절대 경로 출력

pwd

1-2. ls: 현재 디렉토리에 있는 파일 및 디렉토리 목록을 출력

#현재 디렉토리에 있는 파일 및 디렉토리 목록을 출력
ls

#파일의 상세정보와 함께 목록을 출력
ls -l

#숨김 파일을 포함하여 목록을 출력
ls -a 

#숨김 파일을 포함하여 파일의 상세정보와 함께 목록을 출력
ls -la

1-3. cd: 현재 디렉토리에서 다른 디렉토리로 이동

cd {directory_path}     # 특정 디렉토리로 이동
cd ..                   # 상위 디렉토리로 이동
cd ~                    # root(최상위) 디렉토리로 이동

2. 파일 또는 디렉토리 조작 명령어

2-1. mkdir: 새 디렉토리를 생성

mkdir {directory_name}

2-2. rmdir: 빈 디렉토리를 삭제

rmdir {directory_path}

2-3. rm: 파일 혹은 디렉토리를 영구적으로 삭제

# 특정 파일 삭제.
# rm -i {file_name} 명령어와 동일
rm {file_name}

# 특정 파일 삭제.
# 삭제 시 정말로 삭제할 것인지에 관한 확인 메세지를 표시.
rm -i {file_name}

# 특정 파일 삭제.
# 삭제 시 추가적인 확인 메세지를 표시하지 않고 즉시 삭제
rm -f {file_name}


# 특정 디렉토리를 삭제
rm -r {directory_path}
     
# 특정 디렉토리와 해당 디렉토리 내부의 하위 디렉토리를 강제로 전부 삭제
rm -rf {directory_path}

2-4. cp: 파일 또는 디렉토리를 복사

cp {복사할 file_name} {대상 file_name}
cp -r {복사할 directory_path} {대상 directory_path}

2-5. touch: 크기가 0인 새 파일을 생성하거나, 생성된 파일이 존재한다면 파일의 최종 수정 시간을 변경

touch {file_name}

2-6. mv: 파일 또는 디렉토리의 이름을 변경하거나 다른 디렉토리로 이동

# 특정 파일을 대상 디렉토리 내부로 이동
mv {이동할 file_name} {대상 directory_path}


# 여러 특정 파일을 대상 디렉토리 내부로 이동
mv {이동할 file_name_1} {이동할 file_name_2} {대상 directory_path}


# 파일 이름 변경
mv {변경 전 file_name_1} {변경 후 file_name_2}


# 특정 디렉토리를 대상 디렉토리 내부로 이동
mv {이동할 directory_name} {대상 directory_name}

2-7. cat: 파일 내용을 화면에 출력

cat {file_name}

3. 프로세스 관리 명령어

3-1. ps: 현재 실행 중인 프로세스 목록을 표시

ps

3-2. top: 시스템 리소스와 상위 프로세스 정보를 실시간으로 표시

top

3-3. pgrep: 특정 프로세스의 PID를 검색

ps

3-4. sleep: 프로세스를 지정 시간동안 일시정지

sleep {시간}

3-5. kill: 특정 프로세스를 종료

kill {PID}

3-6. pkill: 특정 프로세스 이름 또는 조건에 해당하는 프로세스를 종료

pkill {process_name}

3-7. killall: 특정 프로세스 이름을 가진 모든 프로세스를 종료

killall {process_name}

Memory 관리

메모리 관리가 필요한 이유

  • 프로세스별로 실행 전후에 프로세스 실행에 필요한 만큼의 메모리를 할당하고 해제해야 한다.
  • 여러 프로세스가 메모리를 공유하기 때문에, 프로세스 별로 사용된 메모리 공간을 추척하고 프로세스를 실행하는 동안 데이터 무결성을 유지해야 한다.
  • 설치된 물리적 메모리보다 더 큰 메모리 공간을 필요로 할 때 메모리 용량의 한계를 극복해야한다.

메모리 관리 전략

1. 페이징(Paging)

고정 크기의 페이지로 나누고, 프로세스의 가상 메모리도 같은 크기의 페이지로 나누어 관리한다. 개별 페이지는 순서에 상관없이 물리 메모리에 있는 프레임 블록에 매핑되어 저장한다.

논리 메모리는 고정 크기의 페이지, 물리 메모리는 고정 크기의 프레임 블록으로 나누어 관리하고 CPU에겐 마치 프로세스가 연속된 메모리에 할당된 것처럼 인식하도록 함으로써 메모리 공간이 연속적으로 할당되어야 한다는 제약 조건을 없앤다.

프로세스가 실제 사용해야 할 메모리보다 더 큰 메모리를 할당받아 메모리가 낭비되는 내부 단편화(Internal Fragmentation) 현상이 발생한다.

2. 세그멘테이션(Segmentation)

논리 메모리와 물리메모리를 서로 다른 크기의 논리적 단위인 세그먼트로 분할한다. 페이징 기법과 마찬가지로 개별 페이지는 순서에 상관없이 물리 메모리에 있는 프레임 블록에 저장한다.

메모리에 프로세스가 반복적으로 적재 및 제거되면서 여유 공간이 충분함에도 불구하고 사용 가능한 여유 공간이 조각으로 흩어져있어 프로세스를 메모리에 적재하지 못해 메모리가 낭비되는 외부 단편화(External Fragmentation) 현상이 발생한다.

3. 페이징과 세그멘테이션 혼용 기법

프로세스를 세그먼트로 나눈 뒤, 나누어진 세그먼트를 다시 페이지 단위로 나누어 관리한다.

두 기법을 혼용하여 단편화를 최소화해 메모리 낭비를 줄이기 위한 전략이지만, 두 개의 매핑 테이블을 사용해야 하기 때문에 속도가 감소하는 단점이 있다.

IPC (Inter Process Communication)

IPC (Inter Process Communication)란?

IPC는 프로세스 간 데이터를 주고받는 소통 방법을 말한다. 메세지를 직접 전달하거나 메모리 영역을 공유하는 등의 방법을 통해 상호간 통신한다.

IPC의 주요 방법

  1. PIPE (파이프): 단방향 통신의 파이프를 이용하여 한 프로세스의 출력을 다른 프로세스의 입력으로 직접 연결한다.
  1. 이름 있는 파이프 (Named PIPE): 파일 시스템에 특정 이름을 가진 파이프를 생성하여 두 프로세스가 서로 다른 시간에 실행될 수 있도록 한다.
  1. 메세지 큐 (Message Queue): 프로세스 간에 메세지를 큐에 저장하여 순서대로 전달한다.
  1. 소켓 (Spckets): 클라이언트-서버 모델을 활용해 통신한다.
  1. 공유 메모리 (Shared Memory): 여러 프로세스가 접근 가능한 공유 메모리 영역을 설정하여 프로세스들이 데이터 자체를 공유하도록 지원한다.

I/O 관리

I/O 관리의 필요성

  • 입출력 도중 발생할 수 있는 다양한 오류를 감지해 처리해야 한다.
  • 하드웨어상에서의 복잡한 동작을 숨기고 사용자가 더욱 쉽게 입출력 기능을 사용할 수 있도록 해야 한다.
  • 키보드, 마우스, 화면 등의 다양한 입출력 장치를 해당 장치에 맞춰 관리하여야한다.

입출력 관리 기법

  1. 폴링(Polling):

    CPU가 주기적으로 입출력 장치의 이벤트를 확인하여 작업을 진행하다 입출력 명령을 만나면 직접 입출력 장치에서 데이터를 송수신다.

    입출력이 끝날 때까지 CPU는 다른 작업을 진행할 수 없기 때문에 CPU 자원을 비효율적으로 사용할 수 있다는 문제점이 있다.

  1. 인터럽트(Interrupt):

    입출력 장치에서 CPU로 인터럽트 신호를 전송하여 필요할 때마다 CPU와 통신하여 데이터를 송수신한다.

    입출력이 끝나지 않더라도 CPU를 사용할 수 있지만, 하나의 글자를 처리할 때마다 CPU의 가용 시간을 사용한다는 단점이 있다.

  1. DMA 입출력(Direct MEmory Access):

    시스템 버스에 DMA 컨트롤러라는 하드웨어를 설치한다. 그리고 CPU는 컨트롤러에 입출력 장치의 주소, 수행할 연산 등에 관한 정보를 통해 입출력 작업을 명령한다. DMA 컨트롤러는 직접 장치 컨트롤러와 상호작용하여 입출력 작업을 수행하고, 필요할 경우 메모리에 직접 접근하여 데이터를 읽거나 쓴다. 입출력 작업이 종료되면, DMA 컨트롤러는 CPU에 인터럽트 신호를 전송하여 작업의 종료를 알린다.

    이러한 방식을 통해 CPU를 거치지 않고도 입출력 장치와 메모리가 상호작용 할 수 있다.

  1. 입출력 버스(I/O Bus):

    DMA 컨트롤러와 장치 컨트롤러들을 입출력 버스에 연결하여 사용한다.

    모든 장치가 같은 시스템 버스를 공유한다면, DMA를 위해 한 번 메모리에 점근할 때 메모리에서 DMA 컨트롤러로, DMA 컨트롤러에서 장치 컨트롤러로 데이터를 옮기기 위해 버스를 두 번 사용하게 되는 문제가 있다. 이로 인해 CPU가 버스를 사용하지 못하는 문제를 방지하기 위해 하나의 버스를 추가해 사용한다.

기본 네트워크 개념

OSI 7계층

일반적으로 OSI 7계층이라고 불리는 OSI(Open Systems Interconnection) 모델은 국제 표준화 기구(ISO)에서 개발한 모델로, 네트워크 통신 기능을 7개의 계층으로 나누어 설명한 것이다.

각 단계는 다음과 같이 나누어진다.

  1. 물리계층: 통신 케이블과 같은 물리적 장비를 통해 데이터를 주고받는다.
  1. 데이터 링크 계층: 물리 계층이 이미 존재하는 네트워크를 통해 시스템을 연결하여 해당 네트워크 상의 주변 장치들 간의 데이터를 전송한다.
  1. 네트워크 계층: 여러개의 노드를 거칠 때마다 경로를 찾아주는 역할을 한다. 다양한 길이의 데이터를 네트워크를 통해 전달하고, 그 과정에서 전송 계층이 요구하는 서비스 품질을 제공하기 위한 기능적, 절차적 수단을 제공한다.
  1. 전송 계층: 데이터 패킷이 손실이나 오류 없이 올바른 순서로 도착하게 하고, 필요한 경우 데이터 패킷을 원활하게 복구한다.
  1. 세션 계층: 일대일 애플리케이션 연결 및 동기화 충돌의 시작과 끝을 관리합니다.
  1. 표현(프레젠테이션) 계층: 코드 간의 번역을 담당하여 사용자 시스템에서 데이터의 형식상 차이를 다루는 부담을 응용 계층으로부터 덜어 준다.
  1. 응용(어플리케이션) 계층: 응용 프로세스와 직접 관계하여 일반적인 응용 서비스를 수행한다.

TCP/IP 프로토콜

TCP/IP (Transmission Control Protocol/Internet Protocol)은 인터넷에서 데이터를 전송하기 위해 사용되는 핵심 프로토콜(통신규약) 스위트(Suite)로, 전송 조절 프로토콜인 TCP와 패킷 통신 방식의 인터넷 프로토콜인 IP로 나누어져있다.

7계층으로 나누어진 OSI 모델과는 다르게 총 4개의 계층으로 이루어져 있다.

네트워크 구성 요소

  • 라우터 (Router):
    • LAN(Local Area Network:근거리 통신망)을 연결해 주는 장치로, 서로 다른 네트워크 간의 데이터 전송을 담당한다.
    • 데이터 패킷을 받아 IP주소 기반의 목적지 주소를 확인한 후 라우팅(최적의 경로를 탐색)하여 패킷을 전송한다.
  • 스위치 (Switch)
    • 연결된 장치들의 IP와 MAC 주소를 모두 테이블 형태로 가지고, 원하는 목적지에 데이터 패킷을 전송하는 장치로, 한 네트워크 내의 여러 장치를 연결한다.
    • 데이터 트래픽의 효율성을 높이는 역할을 한다.
  • 허브(Hub)
    • 네트워크 상의 여러 장치를 연결하는 단순한 장치이다.
    • 스위치와는 다르게 연결된 모든 장치들에게 데이터 패킷을 전송한다. 그로인해 네트워크 효율성이 떨어지는 단점이 있다.
