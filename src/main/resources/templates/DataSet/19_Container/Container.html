Docker

Docker는 애플리케이션을 신속하게 구축, 테스트 및 배포할 수 있는 소프트웨어 플랫폼이다. Docker는 소프트웨어를 컨테이너라는 표준화된 유닛으로 패키징하며, 이 컨테이너에는 라이브러리, 시스템 도구, 코드, 런타임 등 소프트웨어를 실행하는 데 필요한 모든 것이 포함되어 있습니다.

컨테이너란?

컨테이너는 가상화 기술 중 하나이다. 기존의 OS를 가상화 시키던것과 달리 컨테이너는 OS레벨의 가상화로 프로세스를 격리시켜 동작하는 방식으로 이루어진다.

가상머신(Virtual Machine) VS 도커 컨테이너(Docker Container)

  • 가상머신
    • 호스트 운영체제 위에 가상화된 하드웨어 계층을 생성하고, 각 가상 머신은 독립된 운영체제, 커널, 드라이버 등을 가진다.
    • 이로 인해 무겁고 높은 자원 소비가 필요하다.
    • 운영체제의 부팅 과정이 필요하므로 시간이 오래 걸린다.
    • 독립된 운영체제를 가지므로 메모리, 디스크 공간 등 자원을 많이 소비된다.
  • Docker 컨테이너
    • Docker 컨테이너는 호스트 운영체제의 커널을 공유하며, 가볍게 격리된 환경을 생성한다.
    • 가상 머신보다 더 가벼우며 효율적으로 실행한다.
    • 이미지와 컨테이너 레이어를 사용하여 빠르게 생성되며, 실행 속도가 매우 빠르다.
    • 컨테이너는 호스트 운영체제의 커널을 공유하므로 가볍고 효율적으로 자원을 활용할 수 있다.

즉, Docker 컨테이너는 기존 가상머신에 비해 가볍고 빠르게 실행되며, 호스트의 운영체제의 커널을 공유하는 방식으로 동작한다. 또한 컨테이너 기술은 애플리케이션 배포와 관리를 더 효율적으로 처리할 수 있는 방법을 제공한다.

Docker의 구성과 동작 순서

Docker는 container를 만들고 배포하고 구동하는 식의 방식으로 진행이 된다. Container를 만들기 위해 먼저 docker file을 만들고 이를 이용하여 image를 만든 뒤에 container를 구동할 수 있다.

  1. Docket File
    Docker file은 container를 어떻게 만들어야 하는지에 대한 설명서와 같은 역할을 한다. 도커 이미지를 만들때 사용하는 파일. docker build 명령어를 실행시키면 도커 이미지를 만들 수 있다.
  1. Docker Image

    컨테이너를 생성할 때 필요한 요소이며, 가상 머신을 생성할 때 사용하는 iso 파일과 비슷한 개념이다.이미지는 컨테이너를 생성하고 실행할 때 읽기 전용으로 사용되며 여러 계층으로 된 바이너리 파일로 존재한다. Docker Image를 docker run 명령어를 실행시키면 Docker Container를 만들 수 있다.
  1. Container

    마지막 단계인 컨테이너는 만들어둔 application의 이미지를 고립된 환경에서 개별적인 file system 안에서 실행할 수 있는 것을 말한다. 즉, 컨테이너 안에서 우리가 만든 appliation이 동작한다고 보면 된다. 컨테이너는 미리 준배해 놓은 application의 이미지를 이용해서 application을 구동하게 되는 것이다.

Docker 사용 이유

  • 환경 일치성: 다양한 환경에서 동일한 실행 환경을 보장한다.
  • 편리한 배포: 이미지로 패키징되어 배포되기 때문에 어플리케이션 배포가 간단하다.
  • 격리된 환경: 컨테이너를 격리된 환경으로 실행하여 하나의 컨테이너에서 문제 발생하여도 다른 컨테이너에 영향을 주지 않는다.
  • 자원 효율성: 가상 머신과 비교해 더 가볍고 빠르며, 호스트 시스템의 리소스를 효율적으로 활용할 수 있다.
  • 스케일링: 컨테이너 기반 아키텍처는 쉬운 스케일링이 가능하여 요구에 따라 응용프로그램을 확장할 수 있다.
