NoSQL 이란?

→ Not Only SQL 이라는 뜻으로, 테이블 형식을 따르지 않는 DB를 뜻한다. NoSQL의 범주로는 DocumentDB(MongoDB), Key-Value DB(Cassandra DB, DynamoDB), Graph DB(페이스북 자체적으로 만든 Tao DB)가 있다.

장점설명
1. 수평적 확장에 유리서비스 규모가 확장되면 개발자는 Scale-up 또는 Scale-out 중 선택해야한다. 개발자는 보통 Scale-out을 선호하고 NoSQL은 Scale-out을 돕는다.
2. 빠른 속도Hash Table 구조를 사용하기 때문에 O(1)의 시간복잡도가 소모된다.
(SQL의 경우 : B-Tree 구조로 O(logN)의 시간복잡도 소모)
3. 유연성Schemaless 로 테이블만 존재하고 스키마가 존재하지 않는다.
또한 Key-Value DB 이므로 비정형 데이터를 저장하기 유리하다.
  • Scale-up : 더 높은 성능을 내는 하드웨어로 교체한다. (하나의 장비에 트래픽이 몰리는 경우 리스크가 커지기 때문에 비선호, Power wall 현상으로 인한 쓰로틀링 현상 발생)
  • Scale-out : 비슷한 성능을 내는 하드웨어를 추가한다.

NoSQL 단점

  1. Join 연산 불가
  1. Primary Key가 아닌 Attribute에 대해서 Conditional Check를 수행할 때는 SQL보다 느리다.

DynamoDB의 2가지 보조 인덱스

  • LSI(Local Secondary Index): 소속된 테이블과 Partition Key가 동일하지만 Sort Key가 다른 인덱스
    (5개까지 할당 가능)
  • GSI(Global Secondary Index): 소속된 테이블과 Partition Key와 Sort Key가 모두 다를 수있는 인덱스
    (20개까지 할당 가능)
  • 보조 인덱스를 이용한 테이블간 연관관계 구현 가능 → Join 가능
